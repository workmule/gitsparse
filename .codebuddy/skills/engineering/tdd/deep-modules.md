# 深模块

源自《软件设计哲学》：

**深模块** = 小接口 + 大量实现

```
┌─────────────────────┐
│     小接口          │  ← 少量方法，简单参数
├─────────────────────┤
│                     │
│                     │
│    深实现           │  ← 隐藏复杂逻辑
│                     │
│                     │
└─────────────────────┘
```

**浅模块** = 大接口 + 少量实现（应避免）

```
┌─────────────────────────────────┐
│          大接口                  │  ← 许多方法，复杂参数
├─────────────────────────────────┤
│    薄实现                       │  ← 只是透传
└─────────────────────────────────┘
```

设计接口时，问：

- 能减少方法数量吗？
- 能简化参数吗？
- 能把更多复杂性隐藏在内部吗？
